GNSS-Aided INS

This project is a custom implementation of a loosely coupled extended Kalman filter. The goal of the filter is to provide better flight data for the other software modules. full-fusion is the primary repository, which features:

  • Loosely coupled extended Kalman filter
  • Flight path visualizer, attitude visualizer, and flight data graphs
  • Testing environment

Another part of this project is the data emulator, which provides sensor data for user inputted flight paths (credit to Aceinna for making their GNSS/INS simulation project open source). I used the emulated data to test my software before deploying it in the plane. Final competition variants of the filter can be found in codex.

The red trace in the map (true path) and the blue trace (path generated with the algorithm from emulated IMU and GPS data) overlap almost perfectly once initial GPS noise is filtered out. The green path is raw GPS data.

Advanced Fuselage

Empennage Attachment

The empennage attachment connects the vertical and horizontal stabilizers to the tailboom. Worst cases for banking and landing were analyzed, tested with SolidWorks FEA, and physically tested.

This is the spreadsheet that I used to predict loading during banking.

Tailboom Bracket

The tail boom brackets are waterjet out of 0.375” aluminum. I designed them in SolidWorks with DFMA methods and conducted FEA to simulate landing loads. They consist of two pieces — one piece is fixed to the center ribs of the fuselage and the other piece can be screwed on to secure the tail boom.
